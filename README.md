<h2>1. 안드로이드 앱 개발 준비하기</h2>
- 안드로이드 버전별 특징<br>
- 안드로이드 개발 준비<br>
<br>
본격적으로 안드로이드 개발을 공부하기 전에 안드로이드가 사용자 관점에서 어떻게 변화했는지를 가볍게 다뤄본다. 또한 안드로이드 앱을 개발하는 환경도 구축해본다.<br>
	<br>
<h2>2. 첫 번째 앱 만들기</h2>
- 안드로이드 프로젝트 만들기<br>
- 프로젝트 구성<br>
- 그레이들 빌드 시스템<br>
- 앱 실행<br>
<br>
간단한 첫 번째 앱을 만들면서 안드로이드 앱 개발의 흐름을 익힌다. 앱을 개발하면서 필요한 기본 개념과 안드로이드 스튜디오의 사용법도 함께 익혀본다. 직접 예제를 만들어보면서 레이아웃을 구성하는 방법과 레이아웃을 자바 클래스에서 불러오는 방법을 배우고, 안드로이드 화면을 구성하는 단위가 무엇이며 화면 간의 이동 방법은 무엇인지 배워보도록 한다.<br>
	<br>
<h2>3. 뷰와 뷰그룹</h2>
- 뷰와 뷰그룹<br>
- 레이아웃 기초<br>
- 간단한 사용자 인터페이스 구성<br>
- 인텐트: 다른 액티비티 실행<br>
<br>
안드로이드에서 화면을 구성하는 기본 요소인 뷰(View)와 뷰 크룹(ViewGroup)의 개념을 학습한다.<br>
	<br>
<h2>4. 자주 사용하는 뷰, 유용한 뷰</h2>
- 자주 사용하는 뷰<br>
- 기타 유용한 뷰<br>
<br>
자주 사용하는 다섯 가지 뷰와 기타 유용한 뷰의 기본적인 사용 예를 살펴본다.<br>
	<br>
<h2>5. 레이아웃: 화면에 뷰를 수놓는 방법</h2>
- LinearLayout (선형 레이아웃)<br>
- RelativeLayout (상대 레이아웃)<br>
- FrameLayout (프레임 레이아웃)<br>
- ConstraintLayout (제약 레이아웃)<br>
	<br>
안드로이드의 레이아웃을 구성하기 위해 필요한 개념인 뷰그룹(ViewGroup)에 대해 배우도록 한다. 대표적인 뷰그룹으로 LinearLayout, RelativeLayout, FrameLayout과 ConstraintLayout이 있다.<br>
<h2>6. 안드로이드는 액티비티로부터</h2>
- 안드로이드의 4대 구성요소<br>
- 화면은 곧 액티비티다<br>
- 액티비티 상속의 변천사와  AppCompatActivity<br>
- 액티비티끼리 데이터 주고받기<br>
	<br>
안드로이드에서 하나의 화면을 구성하는 액티비티와 그와 관련 된 컴포넌트들의 사용방법을 익힌다. 또한 새로운 화면을 시작하거나 간단한 데이터를 주고받을 수 있는 인텐트(Intent)의 개념을 배운다.<br>
<h2>7. 인텐트와 인텐트 필터</h2>
- 인텐트 사용방법<br>
- 명시적 인텐트와 암시적 인텐트<br>
- 전화 걸기 앱 만들기: 암시적 인텐트 사용<br>
- 암시적 인텐트를 사용하는 다양한 예<br>
- 카카오톡에 메시지 전달하기<br>
	<br>
인텐트(Intent)는 일종의 메시지 객체로서 무엇을 하려고 하는지 목적을 가지는 객체라고 할 수 있으며, 이를 이용해 다른 앱 구성 요소로 작업을 요청할 수 있다. 안드로이드 앱 개발에서 인텐트는 주로 새로운 액티비티를 시작하는 용도로 사용한다.<br>
<h2>8. 메뉴 구현하기</h2>
- 액션바(ActionBar)<br>
- 옵션 메뉴<br>
	<br>
메뉴는 앱에서 보편적으로 사용하는 사용자 인터페이스이다. 액션바를 간단하게 다루고, 후반부에서 툴바를 다시 다루도록 한다.
<br>
<h2>9. 웹뷰: 웹 페이지 표시하기</h2>
- 나만의 웹 브라우저 만들기<br>
- 웹 브라우저 보완하기<br>
	<br>
안드로이드에서 웹 페이지를 표시하는 웹뷰(WebView)를 사용해 웹 브라우저를 만들어보고, 사용법을 익혀본다.
<br>
<h2>10. 화면 제약을 극복하는 방법</h2>
- 스크롤뷰(ScrollView)<br>
- 리스트뷰(ListView)와 그리드뷰(GridView)<br>
- 어댑터뷰(AdapterView)<br>
- 나만의 어댑터 만들기: BaseAdapter<br>
- 리스트뷰의 성능 최적화: 뷰홀더 패턴<br>
	<br>
화면에 표시해야 할 내용이 많을 때 스크롤을 지원하는 뷰인 ScrollView와 ListView, GridView에 대하여 알아보도록 한다.
<br>
<h2>11. 기기에 데이터 저장하기: SharedPreference</h2>
- 데이터를 저장하는 간단한 앱<br>
- 기기에 설치된 앱의 정보 이용<br>
- 다이얼로그 추가하기<br>
	<br>
데이터양이 비교적 적은 키(key)-값(value) 구조의 자료를 SharedPreference API를 이용하여 기기에 데이터를 저장해본다.
<br>
<h2>12. 액티비티 생명주기</h2>
- 여러 액티빝티의 관리<br>
- 액티비티 생명주기 소개<br>
- 생명주기 콜백 메서드<br>
- 화면 회전에도 끄떡없게 만들기<br>
- 화면 회전 막기<br>
- 다중 창 막기<br>
	<br>
액티비티는 생성, 중단, 재개, 소멸 등의 생명주기(Life Cycle)가 존재하고, 이는 다양한 환경에서 액티비티를 정상적으로 동작시키기 위해 안드로이드는 각 상태 변화에 따른 생명주기 메서드를 제공하고 있다. 액티비티의 생명주기 메서드의 사용법과 액티비티의 상태를 복원하는 방법에 대해 알아본다.<br><br>
<i>사용자가 앱 실행 중 전화가 오더라도 앱은 일시정지가 되고, 다시 이어서 동작을 한다. 이는 액티비티의 생명주기에 맞추어 적절히 코딩이 되었기 때문이다. 또는 기기를 회전시켰을 때 레이아웃이 화면 크기에 맞게 변경되는 것도 액티비티의 생명주기 메서드와 연관이 깊다.</i>
<br>
<h2>13. 프래그먼트: 화면을 조각조각 나누기</h2>
- 프래그먼트 소개<br>
- 프래그먼트 실습<br>
- 다이얼로그 프래그먼트(DialogFragment)<br>
- 프래그먼트와 액티비티 간의 통신(콜백 구현하기)<br>
		<br>
간단히 컨텐츠와 생명주기를 가지는 뷰로 설명할 수 있는 프래그먼트(Fragment)를 이용해 각 레이아웃의 부분을 따로 만들어두고 하나의 액티비티에서 조합하는 형태로 개발할 수 있게 되었다. 프래그먼트를 이해하는데 필요한 이론과 실습, 프래그먼트와 액티비티가 서로 통신하는 방법을 알아보도록 한다.
<br>
<h2>14. 뷰페이지: 좌우로 밀리는 화면</h2>
- 뷰페이저 구현 실습<br>

<h2>16. 리소스</h2>

	<br>
안드로이드 리소스(resource)의 종류와 활용하는 방법을 배워본다.
<h2>17. 브로드캐스트 리시버</h2>
- 미리 정의된 방송<br>
- 사용자 정의 방송<br>
- 브로트캐스트 리시버의 특징<br>
	<br>
안드로이드 장치에서 발생하는 이벤트/정보에 반응하는 컴포넌트인 브로드캐스트(BroadCast)를 알아보도록 한다. 브로드캐스트(BroadCast)가 인텐트(Intent)를 통해 발송되면 브로드캐스트 리시버(BroadCast Receiver) 객체가 수신하게 된다.<br><br>
<i>브로드 캐스트 종류에는 앱의 시작/삭제/재설치... 또는 배터리 양의 변화 외에 다양한 표준 브로드캐스트들이 존재하고, 개발자가 직접 추가할 수도 있다.</i>
<br>

<br>
<h2>19. 비동기 처리</h2>

	<br>
안드로이드에서는 비동기 처리를 위해 자바의 스레드(Thread)와 안드로이드의 핸들러(Handler)를 활용하거나 AsyncTask 클래스를 사용하는 방식이 있다. 안드로이드의 대표적인 비동기 처리 패턴을 알아보도록 한다.
